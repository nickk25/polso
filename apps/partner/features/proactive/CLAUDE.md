# features/proactive

Proactive client messaging (receipt reminders, summaries, anomaly alerts, bank-reconnect nudges) sent via Telegram/WhatsApp through @polso/agent; triggered by the `/api/cron/daily` cron and by partner-initiated actions.

## Files

- `actions/send-reminder.ts` — partner-triggered receipt reminder for one client; checks `autoRemindersEnabled` + `reminderCooldownHours` cooldown before delegating to `sendReminderInternal`.
- `actions/send-bulk-reminder.ts` — reminders for all clients or only those with <100% documented transactions this month; returns sent/skipped counts with reason codes.
- `actions/send-receipt-request.ts` — AI message listing specific transactions missing attachments (`receipt_request_list`).
- `actions/send-bank-reconnect.ts` — AI nudge to reconnect an expired/erroring bank account (`bank_reconnect`).
- `lib/evaluate-triggers.ts` — cron trigger priority: monthly summary (day 1) > weekly summary (Monday) > anomaly alert > receipt reminder; each gated by a was-sent-within window check on ProactiveMessage.
- `lib/gather-context.ts` — builds `ProactiveContext` payloads: unmatched transactions (current quarter, `transactionNotDocumentedWhere`), weekly/monthly summaries, anomalies (`detectAnomalies` from @polso/intelligence) and missing monthly recurring charges.
- `lib/send-proactive-message.ts` — channel resolution (prefers Telegram — no 24h session window), global 1-message/24h/org rate limit, send + log to ProactiveMessage.
- `lib/send-reminder-internal.ts` — shared receipt-reminder pipeline (resolve channel, gather unmatched, generate AI message, send, log) used by single and bulk actions.
- `components/send-reminder-button.tsx` — client button with cooldown-aware disabled state and "last contacted" label; compact icon variant for tables.

## Key flows

- All server actions verify `orgType === "partner"` and the active `PartnerClient` link before touching client data.
- Telegram chat ID lives on `UserOrganization.telegramChatId` (first linked member); WhatsApp phone on `Organization.whatsappPhone`. Telegram wins when both exist.
- Message content is AI-generated by `generateProactiveMessage` (@polso/agent/proactive, Claude Haiku); the cron additionally checks `checkAiRateLimit` (@polso/cache).
- Manual actions can still fail with "límite de 24h" because `sendProactiveMessage` enforces the global per-org rate limit (except `sendReminderInternal`, which only respects the partner's cooldown via `send-reminder.ts`).
- Cron passes the partner's `receiptReminderHours` (default 48) as the receipt-reminder dedupe window and skips clients whose partner disabled `autoRemindersEnabled`.
